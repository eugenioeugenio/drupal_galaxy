 # Учебный проект: Развертывание Drupal 11 с помощью Ansible и Vagrant

Этот проект автоматизирует создание инфраструктуры и установку системы управления контентом Drupal на базе Ubuntu 22.04.

## Структура проекта
- `Vagrantfile`: Конфигурация виртуальной машины (VirtualBox, статический IP 192.168.56.10).
- `inventory`: Файл со списком хостов и параметрами подключения по SSH.
- `ansible.cfg`: Глобальные настройки Ansible для упрощения запуска.
- `playbook.yml`: Основной сценарий установки (LAMP-стек + Drupal).
- `vars/default.yml`: Переменные конфигурации (версии PHP, БД, домены).
- `execution_log.txt`: Лог успешного выполнения сценария.

## Системные требования
- Windows с поддержкой WSL2 (Ubuntu).
- VirtualBox 7.x.
- Vagrant 2.4.x.
- Ansible (установлен внутри WSL).

## Инструкция по запуску
1.  Поднять виртуальную машину: `vagrant up` ( из папки demovm), она в  домашнем каталоге как и drupal_galaxy
2.  Перейти в папку проекта: `cd drupal_galaxy`
3.  Установить необходимые роли из Ansible Galaxy:
   `ansible-galaxy install -r requirements.yml` (или перечислить роли вручную) - в данном проекте последнее.
4.  Запустить установку: `ansible-playbook playbook.yml`
5.  После завершения сайт доступен по адресу: `http://192.168.56.10`

## Особенности реализации
- Использованы проверенные роли от Geerlingguy.
- Исправлены ошибки обратной совместимости старых ролей с современным Ansible (Boolean-заплатки в vars).
- Настроена синхронизация через rsync для корректной работы в WSL2.
